### Data Flow: Scoring → Frontend

- `ScoringRehydrationController` → REST API `/api/scoring/conversation/:id`
- Frontend calls `fetchScoringResult()` from `apps/web/src/lib/api/scoring.ts`
- Returns `ScoringCompletePayload['result']` which includes `dimensionScores[].findings`
- **Findings JSONB already contains `assessmentConfidence` and `isoClauseReferences` from Epic 37**
- Frontend `@/types/scoring.ts` has `DimensionScoreData` type — needs ISO field additions

### Data Flow: Scoring → Export

- `ScoringExportService.getScoringData()` fetches from DB repositories
- Passes `ScoringExportData` to PDF/Word exporters
- `ScoringExportData` currently: `{ report, vendorName, solutionName, generatedAt }`
- **Does NOT include raw dimension scores with ISO/confidence data** — needs enrichment

## 3. Pattern Verification

### Correct Assumptions

| Assumption | Evidence |
|------------|----------|
| ScoringExportService supports PDF/Word | `exportToPDF()`, `exportToWord()` methods confirmed |
| ISO fields in scoring output | `dimensionScores.findings` JSONB has `assessmentConfidence` + `isoClauseReferences` (Epic 37) |
| HTML template for PDF | `scoring-report.html` confirmed |
| Frontend has assessment detail view | ScoringResultCard + ScoreDashboard confirmed |

### Incorrect/Incomplete Assumptions

| Assumption | Reality |
|------------|---------|
| Excel export for scoring exists | **NO.** Only questionnaire Excel. Must create `ScoringExcelExporter` + interface. |
| Narrative prompt has ISO access | **NO.** `exportNarrativePrompt.ts` has zero ISO references. Must inject. |
| Frontend types mirror backend | **PARTIAL.** `@/types/scoring.ts` has `DimensionScoreData` but no ISO/confidence fields yet. |

### Patterns to Follow

| Pattern | Example | Where |
|---------|---------|-------|
| Export interface + implementation | `IScoringPDFExporter` → `ScoringPDFExporter` | New Excel exporter should follow same |
| Template variable injection | `{{dimensionScores}}` in HTML template | Add ISO/confidence variables same way |
| Word programmatic building | `createDimensionTable()` in ScoringWordExporter | Add ISO columns same pattern |
| Frontend Zustand store | `chatStore.ts` with typed state | ISO data flows through existing scoring state |
| Component composition | ScoreDashboard → DimensionScoreBar | New ConfidenceBadge component same pattern |
| Data-testid convention | `data-testid="score-dashboard"` | All new components must have testids |

### Dependencies Not Mentioned in Goals

1. **Frontend types file** (`apps/web/src/types/scoring.ts`) — must add ISO/confidence fields
2. **ScoringExportData type** — must be enriched with dimension-level ISO/confidence data
3. **3 files over 300 LOC** — must split BEFORE adding ISO features (Sprint 1 blocker)
4. **No `IScoringExcelExporter` interface** — must create in application layer
5. **Container.ts** (DI) — must wire new ScoringExcelExporter (recently extracted from index.ts)
