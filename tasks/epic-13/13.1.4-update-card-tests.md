# Story 13.1.4 - Update QuestionnairePromptCard Tests

## Status: Not Started

## Context
Stories 13.1.1 and 13.1.2 changed the QuestionnairePromptCard component:
- Restyled from Shadcn Card to assistant bubble
- Removed dismiss button and `onDismiss` prop

The tests need to be updated to reflect these changes.

## Goal
Update unit tests to match the new component implementation.

## File to Modify
`apps/web/src/components/chat/__tests__/QuestionnairePromptCard.test.tsx`

## Current Test Coverage
The existing tests cover:
1. Ready state rendering (summary, badge, buttons)
2. Generating state (spinner, disabled button, opacity)
3. Download state (format buttons, callbacks)
4. Error state (error message, retry button)
5. Assessment type styling (quick/comprehensive/category_focused)
6. forwardRef functionality
7. Custom className support
8. **Dismiss button tests** (NEEDS REMOVAL)

## Required Changes

### 1. Remove Dismiss Button Tests
Find and remove all tests related to the dismiss button:

```tsx
// REMOVE tests like:

  // ...
});

it('should render dismiss button', () => {
  // ...
});
```

### 2. Remove onDismiss from Test Props
Update the default props helper to not include `onDismiss`:

**Before:**
```tsx
const defaultProps = {
  payload: mockPayload,
  uiState: 'ready' as QuestionnaireUIState,
  onGenerate: jest.fn(),
  onDismiss: jest.fn(),  // <-- REMOVE
  onDownload: jest.fn(),
  onRetry: jest.fn(),
};
```

**After:**
```tsx
const defaultProps = {
  payload: mockPayload,
  uiState: 'ready' as QuestionnaireUIState,
  onGenerate: jest.fn(),
  onDownload: jest.fn(),
  onRetry: jest.fn(),

```

### 3. Update Styling Tests
If there are tests for Shadcn Card classes, update them for bubble styling:

**Before:**

**After:**
