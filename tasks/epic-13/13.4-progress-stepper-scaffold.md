## Requirements
- Implement vertical stepper UI per Appendix demo (context/generating/validating/saving)
- Include animations: fade/slide per-step, pulse on active step, line fill for completed
- Collapsible panel: expands while running, auto-collapses when complete, user can toggle
- Show download buttons when "complete" (use existing handlers)
- Simulate step progression with local state (will be replaced by backend events in 13.5)
- Maintain existing selectors/data-testid where possible

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Stepper location | Inside generating state | Replaces text content, keeps button below |
| Collapse behavior | Per-conversation | State resets on conversation switch |
| Auto-collapse delay | 800ms | Matches Appendix demo, feels natural |
| Step simulation | Timer-based (temporary) | Demonstrates UI before backend wiring |
| Stepper component | Separate file | Reusable, testable in isolation |
| State scoping | Reset on switch (not per-convo map) | Matches existing patterns, simpler |
| Performance | React.memo on VerticalStepper | Prevents re-renders during animation |

## Robustness & Performance

| Area | Implementation | Rationale |
|------|----------------|-----------|
| **State scoping** | Reset `currentGenerationStep` to -1 on conversation switch | Prevents cross-conversation bleed |
| **Timer cleanup** | Clear on unmount, switch, error | Prevents orphaned state updates |
| **Conversation gating** | Check `activeConversationId` in interval | Stops simulation if user switches mid-generation |
| **Memoization** | `React.memo` on VerticalStepper | Prevents unnecessary re-renders |
| **Viewport safety** | `max-w-md` + test at 320px | Ensures bubble doesn't overflow on mobile |
