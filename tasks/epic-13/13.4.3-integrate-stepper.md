# Story 13.4.3 - Integrate Stepper into QuestionnairePromptCard

## Status: Not Started

## Context
Stories 13.4.1 and 13.4.2 created the VerticalStepper component and added state to the store. This story integrates the stepper into QuestionnairePromptCard during the generating state.

## Goal
Embed VerticalStepper in the generating state, replacing the text content section while keeping the generate button.

## File to Modify
`apps/web/src/components/chat/QuestionnairePromptCard.tsx`

## Implementation

### Add Imports
```typescript
import { VerticalStepper } from './VerticalStepper';
import { useChatStore, GENERATION_STEPS } from '@/stores/chatStore';
```

### Get Stepper State
```typescript
// Inside component
const { currentGenerationStep, generationSteps } = useChatStore(
  (state) => ({
    currentGenerationStep: state.currentGenerationStep,
    generationSteps: state.generationSteps,
  })
);
```

### Modify Generating State Section
Replace the content section (approximately lines 195-261) during generating state:

**Current structure:**
```typescript
{/* Ready/Generating state content */}
<div className="flex flex-col space-y-2 mt-3">
  {/* Badge */}
  {/* Vendor name */}
  {/* Context summary */}
  {/* Question count or categories */}
</div>
```

**New structure:**
```typescript
{isGenerating ? (
  // Stepper replaces content during generation
  <div className="mt-3">
    <VerticalStepper
      steps={generationSteps}
      currentStep={currentGenerationStep}
      isRunning={isGenerating}
    />
  </div>
) : (
  // Original ready state content
  <div className="flex flex-col space-y-2 mt-3">
    {/* Badge */}
    {/* Vendor name */}
    {/* Context summary */}
    {/* Question count or categories */}
  </div>
)}
```

### Keep Unchanged
- Ready state header/badge (above the conditional)
- Generate button (below the conditional)
- Download state layout (separate branch)
- Error state layout (separate branch)
- Icon in header (Loader2 spinner already shows during generating)

### UX Constraints
- Bubble respects `max-w-md` to match existing card width
- Collapsible section doesn't push composer on small viewports
- Test at mobile widths (320px, 375px) to verify no overflow

## Acceptance Criteria
- [ ] Stepper visible during `generating` state (uiState === 'generating')
- [ ] Ready state still shows vendor info + button (no stepper)
- [ ] Download state unchanged (no stepper)
- [ ] Error state unchanged (no stepper)
- [ ] Stepper receives correct props from store
- [ ] Generate button still visible below stepper (disabled during generation)
- [ ] **Bubble respects chat width** (max-w-md, no overflow)
- [ ] **Works on small viewports** (320px+)
- [ ] No TypeScript errors
- [ ] No visual regressions in ready/download/error states
