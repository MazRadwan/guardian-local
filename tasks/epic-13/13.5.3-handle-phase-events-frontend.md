# Story 13.5.3 - Handle Phase Events in Frontend

## Status: Not Started

## Context
Backend now emits `generation_phase` events (13.5.2). This story adds frontend listeners to update the stepper state.

## Goal
Listen for `generation_phase` events and update `currentGenerationStep` in the chatStore, with idempotency and conversation scoping.

## Files to Modify
- `apps/web/src/hooks/useWebSocketEvents.ts`
- `apps/web/src/stores/chatStore.ts` (verify actions exist from 13.4.2)

## Implementation

### Add Import
```typescript
import { GenerationPhasePayload } from '@guardian/shared';
```

### Add Handler in useWebSocketEvents.ts
```typescript
// Handler for generation phase events
const handleGenerationPhase = useCallback((data: GenerationPhasePayload) => {
  // Only process for active conversation
  if (data.conversationId !== activeConversationId) {
    console.log('[WS] Ignoring phase event for inactive conversation:', data.conversationId);
    return;
  }

  // Only advance forward (idempotency - handles reconnection/duplicates)
  const currentStep = useChatStore.getState().currentGenerationStep;
  if (data.phase <= currentStep) {
    console.log('[WS] Ignoring out-of-order phase event:', data.phase, '<=', currentStep);
    return;
  }

  console.log('[WS] Generation phase:', data.phaseId, '(', data.phase, ')');
  setCurrentGenerationStep(data.phase);
}, [activeConversationId, setCurrentGenerationStep]);
```

### Register Listener
```typescript
// In the useEffect that registers socket listeners:
```

## Idempotency Logic
```
Current: 0 → Event: 1 → Accept (1 > 0) → Set to 1
Current: 2 → Event: 1 → Reject (1 <= 2) → Stay at 2
Current: 2 → Event: 2 → Reject (2 <= 2) → Stay at 2
Current: -1 → Event: 0 → Accept (0 > -1) → Set to 0
```
