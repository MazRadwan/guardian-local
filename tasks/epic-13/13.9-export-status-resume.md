# Epic 13.9: Persistent Export Ready on Resume

Goal: When a user returns to a conversation on any device/browser, show download buttons without re-generating the questionnaire, while still allowing explicit regeneration.

## Scope
- Server-side export-status check on resume/activation.
- Client resume flow that requests export status and reuses existing `export_ready` handling.
- Preserve current localStorage fast-path; server response is the authoritative fallback.
- Keep regeneration as an explicit user action.

## Stories

### 13.9.1 Backend: Export Status Query (Socket or REST)
- Add a backend entry point (socket event `get_export_status` or lightweight REST) that:
  - Accepts `conversationId` (and user auth).
  - Resolves linked `assessmentId`.
  - Verifies questions exist or assessment status indicates generated.
  - Emits/returns `export_ready`-shaped payload `{ conversationId, assessmentId, questionCount, formats }`.
- Reject with auth/ownership checks and clear error messaging.
- Tests: unit for handler/service; integration for happy path and missing/unauthorized cases.

### 13.9.2 Frontend: Request Export Status on Resume
- On connection ready or conversation switch, request export status from the server if no in-memory or localStorage export data is present.
- Handle the response by reusing the existing `export_ready` flow (store cache, UI -> download, stepper complete).
- Ensure idempotency: do nothing if export is already cached.
- Tests: unit for event handler/request logic; update existing websocket adapter tests.

### 13.9.3 Frontend: UX Guards and Regenerate Option
- If export exists, show download state by default; present a secondary “Regenerate” action that triggers the normal generate flow.
- Ensure errors from export-status request surface a retry path without flipping to generate state.
- Tests: component/rendering states for download-default + regenerate secondary; interaction tests.

### 13.9.4 E2E: Resume and Download Without Regeneration
- Scenario: user generates once, leaves, returns (new session/device), sees download buttons immediately, and successfully downloads without regenerating.
- Include negative path: missing export -> falls back to generate flow.
- Update playwright/QA coverage.

### 13.9.5 Observability and Logging
- Add concise logging around export-status requests/responses and authorization failures.
- Ensure no sensitive data in logs.
- Tests: log presence via unit/mocked assertions where applicable.
