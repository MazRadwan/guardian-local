# Story 13.9.3 - Frontend: UX Guards and Regenerate Option

## Status: Deferred

## Deferral Note
**Deferred:** 2025-12-08 | **Reason:** Over-engineering for current sprint scope.

The primary goal of Epic 13.9 is download persistence across sessions (13.9.1 + 13.9.2). The regenerate option adds UI complexity (confirmation modal, destructive action warnings) without addressing core user needs.

**Current behavior if user clicks "Generate Report" again:** Overwrites existing export. This is an intentional user action and acceptable for MVP.

**When to revisit:**
- If user feedback indicates confusion about overwriting
- If billing/cost concerns require explicit confirmation gates
- Next sprint focused on polish/guardrails

**Dependencies removed from:** 13.9.4, 13.9.5 (they no longer require this story)

---

## Context
When export exists, users see download buttons by default. They should also have an option to regenerate the questionnaire if needed (e.g., requirements changed).

**IMPORTANT:** Regeneration is destructive (deletes current questionnaire) and billable (costs API credits). A confirmation modal is required.

## Goal
Add a "Regenerate questionnaire" option in download state that:
1. Shows a confirmation modal
2. Clears all export data on confirmation
3. Returns to ready state

// In ChatState interface:
clearExportData: (conversationId: string) => void;

// In store implementation:
clearExportData: (conversationId) => {
  set((state) => {

### 3. `apps/web/src/components/chat/ChatInterface.tsx`

Add regenerate handler and pass to QuestionnairePromptCard:

**IMPORTANT:** Must also clear `questionnaireReadyPayload` to prevent 13.9.2 resume logic from immediately restoring the export.

```typescript
// Add handler (near other handlers like handleDownload):
const handleRegenerate = useCallback(() => {
  if (!activeConversationId) return;

  console.log('[ChatInterface] Regenerating questionnaire for:', activeConversationId);

  // Clear export data from store
  useChatStore.getState().clearExportData(activeConversationId);

  // Clear questionnaireReadyPayload to prevent resume logic from restoring
  useChatStore.getState().setQuestionnaireReadyPayload(null);

  // Reset UI state to ready
  useChatStore.getState().setQuestionnaireUIState('ready');

  // Reset generation step
  useChatStore.getState().resetGenerationStep();

  // Clear localStorage
  localStorage.removeItem(`export_${activeConversationId}`);
}, [activeConversationId]);

// Pass to QuestionnairePromptCard:
<QuestionnairePromptCard
  // ... existing props
  onRegenerate={handleRegenerate}
/>
```
