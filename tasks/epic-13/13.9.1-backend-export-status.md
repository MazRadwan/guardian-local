# Story 13.9.1 - Backend: Export Status Query

## Status: Not Started

## Context
When users return to a conversation, they need to see download buttons without regenerating. This story adds a backend socket event that checks if an export already exists for a conversation.

## Goal
Add `get_export_status` socket handler that returns export data if a questionnaire was already generated for the conversation.

## Logging Note
**MVP:** `console.log` / `console.error` is acceptable.

**Recommendation:** Use a logger wrapper to allow future swap to structured logging. See Story 13.9.5 for logger interface suggestion. Ensure no PII in log messages (user IDs and conversation IDs are safe, but no passwords/tokens/content).

```typescript
/**
 * Handle export status query (Story 13.9.1)
 * Returns existing export if questionnaire was already generated.
 * Made async to support repository calls.
 */
async handleGetExportStatus(
  socket: AuthenticatedSocket,
  data: { conversationId: string }
): Promise<void> {
  const { conversationId } = data;
  const userId = socket.data.userId;

  // Early validation: conversationId must be a non-empty string
  if (!conversationId || typeof conversationId !== 'string' || conversationId.trim() === '') {
    console.log(`[ChatServer] get_export_status invalid input: conversationId=${conversationId}`);
    socket.emit('export_status_error', {
      conversationId: conversationId ?? '',
      error: 'Invalid conversation ID'
    });
    return;
  }

  console.log(`[ChatServer] get_export_status request: conversationId=${conversationId}, userId=${userId}`);

  try {
    // 1. Verify conversation ownership
    const conversation = await this.conversationRepository.findById(conversationId);
    if (!conversation) {
      console.log(`[ChatServer] export_status auth error: conversationId=${conversationId}, reason=Conversation not found`);
      socket.emit('export_status_error', {
        conversationId,
        error: 'Conversation not found'
      });
      return;
    }
    if (conversation.userId !== userId) {
      console.log(`[ChatServer] export_status auth error: conversationId=${conversationId}, reason=Unauthorized`);
      socket.emit('export_status_error', {
        conversationId,
        error: 'Unauthorized'
      });
      return;
    }

    // 2. Check for linked assessment
    const assessment = await this.assessmentRepository.findByConversationId(conversationId);
    if (!assessment) {
      console.log(`[ChatServer] export_status not found: conversationId=${conversationId}`);
      socket.emit('export_status_not_found', { conversationId });
      return;
    }

    // 3. Verify questions exist (safe numeric conversion for Drizzle bigint)
    const rawCount = await this.questionRepository.countByAssessmentId(assessment.id);
    const questionCount = typeof rawCount === 'bigint' ? Number(rawCount) : parseInt(String(rawCount), 10);

    if (isNaN(questionCount) || questionCount === 0) {
      console.log(`[ChatServer] export_status not found: conversationId=${conversationId}`);
      socket.emit('export_status_not_found', { conversationId });
      return;
    }

    // 4. Emit export_ready-shaped payload (reuses existing frontend handler)
    socket.emit('export_ready', {
      conversationId,
      assessmentId: assessment.id,
      questionCount,
      formats: ['word', 'pdf', 'excel'],
    });

    console.log(`[ChatServer] export_status found: assessmentId=${assessment.id}, questions=${questionCount}`);

  } catch (error) {
    console.error(`[ChatServer] get_export_status error:`, error);
    socket.emit('export_status_error', {
      conversationId,
      error: 'Internal server error'
    });
  }
}
```

**IMPORTANT:** Drizzle returns `count(*)` as bigint. Use explicit cast or safe parsing:

```typescript
async countByAssessmentId(assessmentId: string): Promise<number> {
  const result = await this.db
    .select({ count: sql<number>`count(*)::int` })  // Cast to int in SQL
    .from(questions)
    .where(eq(questions.assessmentId, assessmentId));

  // Belt-and-suspenders: parse safely even if bigint slips through
  const rawCount = result[0]?.count ?? 0;
  return typeof rawCount === 'bigint' ? Number(rawCount) : parseInt(String(rawCount), 10) || 0;
}
```

## Events Summary

| Event | Direction | Payload | When |
|-------|-----------|---------|------|
| `get_export_status` | Client → Server | `{ conversationId }` | Request |
| `export_ready` | Server → Client | `{ conversationId, assessmentId, questionCount, formats }` | Found |
| `export_status_not_found` | Server → Client | `{ conversationId }` | No export |
| `export_status_error` | Server → Client | `{ conversationId, error }` | Auth/error |
