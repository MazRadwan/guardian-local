# Story 13.4.2 - Add Stepper State to chatStore

## Status: Not Started

## Context
The VerticalStepper component (13.4.1) needs state to track current step and step definitions. This story adds that state to the chatStore.

## Goal
Add stepper-related state and actions to chatStore for tracking generation progress.

## File to Modify
`apps/web/src/stores/chatStore.ts`

## Implementation

### Import Step Type
```typescript
import { Step } from '@/components/chat/VerticalStepper';
// Or define locally if preferred
```

### Define Default Steps
```typescript
// At module level (outside store)
export const GENERATION_STEPS: Step[] = [
  { id: 'context', label: 'Context gathered' },
  { id: 'generating', label: 'Generating questions' },
  { id: 'validating', label: 'Validating structure' },
  { id: 'saving', label: 'Saving assessment' },
];
```

### Add State
```typescript
interface ChatState {
  // ... existing state ...

  // Stepper state
  generationSteps: Step[];
  currentGenerationStep: number;  // -1 = idle, 0-N = in progress, >= length = complete
}
```

### Add Initial Values
```typescript
const initialState: ChatState = {
  // ... existing initial state ...

  generationSteps: GENERATION_STEPS,
  currentGenerationStep: -1,
};
```

### Add Actions
```typescript
interface ChatActions {
  // ... existing actions ...

  setCurrentGenerationStep: (step: number) => void;
  resetGenerationStep: () => void;
}

// In store creation:
setCurrentGenerationStep: (step) => set({ currentGenerationStep: step }),
resetGenerationStep: () => set({ currentGenerationStep: -1 }),
```

### Reset on Conversation Change
Modify existing conversation change logic to reset stepper:
```typescript
// In setActiveConversationId or equivalent
setActiveConversationId: (conversationId) => set((state) => ({
  activeConversationId: conversationId,
  // ... existing resets ...
  currentGenerationStep: -1,  // Reset stepper on conversation change
})),
```

## State Flow

```
Idle:       currentGenerationStep = -1
Step 1:     currentGenerationStep = 0
Step 2:     currentGenerationStep = 1
Step 3:     currentGenerationStep = 2
Step 4:     currentGenerationStep = 3
Complete:   currentGenerationStep = 4 (>= length)
```

## Acceptance Criteria
- [ ] `GENERATION_STEPS` constant exported from store
- [ ] `generationSteps` available in store state
- [ ] `currentGenerationStep` tracks progress (-1 to N)
- [ ] `setCurrentGenerationStep(step)` action works
- [ ] `resetGenerationStep()` sets currentGenerationStep to -1
- [ ] Step resets to -1 when conversation changes
- [ ] TypeScript types correct
- [ ] No breaking changes to existing store consumers
