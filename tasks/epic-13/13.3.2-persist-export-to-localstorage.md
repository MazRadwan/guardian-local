# Story 13.3.2 - Persist Export State to localStorage

## Status: Not Started

## Context
The `exportReadyByConversation` cache in chatStore is in-memory only. After a page reload, export state is lost and users cannot download their questionnaire without regenerating. This story adds localStorage persistence for export state, similar to how payloads are persisted.

## Goal
Persist export state to localStorage so downloads survive page reload, not just conversation switches.

## Files to Modify
1. `apps/web/src/hooks/useQuestionnairePersistence.ts` - Add export persistence methods
2. `apps/web/src/hooks/useWebSocketEvents.ts` - Save export on `export_ready` event
3. `apps/web/src/components/chat/ChatInterface.tsx` - Load export on rehydration

## Required Changes

### 1. Add Export Persistence Methods to useQuestionnairePersistence.ts

```typescript
const EXPORT_KEY_PREFIX = 'guardian_q_export_';

// Add to returned object:
saveExport: (conversationId: string, payload: ExportReadyPayload) => {
  try {
    window.localStorage.setItem(
      `${EXPORT_KEY_PREFIX}${userId}:${conversationId}`,
      JSON.stringify(payload)
    );
  } catch (e) {
    console.error('Failed to save export:', e);
  }
},

loadExport: (conversationId: string): ExportReadyPayload | null => {
  try {
    const data = window.localStorage.getItem(
      `${EXPORT_KEY_PREFIX}${userId}:${conversationId}`
    );
    if (!data) return null;

    const parsed = JSON.parse(data);
    // Shape validation: conversationId match + required fields
    if (parsed.conversationId === conversationId &&
        parsed.assessmentId &&
        Array.isArray(parsed.formats)) {
      return parsed;
    }
    // Malformed entry - clear to prevent state poisoning
    window.localStorage.removeItem(`${EXPORT_KEY_PREFIX}${userId}:${conversationId}`);
    return null;
  } catch (e) {
    console.error('Failed to load export:', e);
    return null;
  }
},

clearExport: (conversationId: string) => {
  try {
    window.localStorage.removeItem(
      `${EXPORT_KEY_PREFIX}${userId}:${conversationId}`
    );
  } catch (e) {
    console.error('Failed to clear export:', e);
  }
},
```

### 3. Save Export in handleExportReady (useWebSocketEvents.ts)

// In handleExportReady, after setExportReady:
const handleExportReady = useCallback((data: ExportReadyPayload) => {
  if (data.conversationId !== activeConversationId) return;

  setExportReady(data.conversationId, data);
  setQuestionnaireUIState('download');

  // Persist to localStorage for page reload survival
  if (persistence) {

### 4. Update Rehydration in ChatInterface.tsx

Modify the rehydration effect to check localStorage export before in-memory:

```typescript
useEffect(() => {
  if (prevConversationIdRef.current === activeConversationId) return;
  prevConversationIdRef.current = activeConversationId;
  if (isGeneratingQuestionnaire) return;

  clearPendingQuestionnaire();
  setQuestionnaireUIState('hidden');

  // Priority 1: Check in-memory export state (conversation switch)
  const exportData = useChatStore.getState().exportReadyByConversation[activeConversationId];
  if (exportData && exportData.conversationId === activeConversationId) {
    setQuestionnaireUIState('download');
    return;
  }

  // Priority 1.5: Check localStorage export (page reload case)
  const savedExport = persistence.loadExport(activeConversationId);
  if (savedExport && savedExport.conversationId === activeConversationId) {
    // Restore to in-memory cache and set UI state
    setExportReady(activeConversationId, savedExport);
    setQuestionnaireUIState('download');
    return;
  }

  // Priority 2: Check localStorage payload
  const savedPayload = persistence.loadPayload(activeConversationId);
  if (savedPayload) {
    if (savedPayload.conversationId === activeConversationId && savedPayload.assessmentType) {
      setPendingQuestionnaire(savedPayload);
      setQuestionnaireUIState('ready');
    } else {
      persistence.clearPayload(activeConversationId);
    }
  }
}, [activeConversationId, user?.id]);
```

## State Priority Order (After Change)

| Priority | Source | Result State | When |
|----------|--------|--------------|------|
| 1 | In-memory `exportReadyByConversation` | 'download' | Conversation switch |
| 1.5 | localStorage export | 'download' | Page reload |
| 2 | localStorage payload | 'ready' | questionnaire_ready, no export |
| 3 | Neither | 'hidden' | No questionnaire activity |

## Security Considerations

- **User-scoped keys**: `guardian_q_export_{userId}:{conversationId}` prevents cross-user leaks
- **Shape validation**: Check conversationId match, assessmentId, and formats array
- **Malformed cleanup**: Remove invalid entries automatically
- **try/catch wrapping**: All localStorage operations wrapped for quota/permission errors

## Security & Validation Guards

| Guard | Location | Purpose |
|-------|----------|---------|
| **Conversation match** | loadExport | Verify export.conversationId === requested conversationId |
| **Shape validation** | loadExport | Check required fields: assessmentId, formats array |
| **Malformed cleanup** | loadExport | Auto-clear poisoned localStorage entries |
| **User-scoped keys** | saveExport/loadExport | Keys include userId to prevent cross-user leaks |
| **Fresh event wins** | handleExportReady | export_ready overwrites both store AND localStorage |
| **Active-convo gating** | rehydration effect | Only restore for activeConversationId |
