# Story 13.6.2 - Add Conversation-Scoped Collapse Reset

## Status: Complete

## Context
The `isStepperExpanded` state is local to QuestionnairePromptCard. When switching conversations, the collapse state should reset to prevent cross-conversation bleed.

## Goal
Reset collapse state when conversation changes AND clear any pending collapse timers. Ensures each conversation starts fresh with proper timer hygiene.

## File to Modify
`apps/web/src/components/chat/QuestionnairePromptCard.tsx`

## Implementation

### Conversation ID Tracking and Timer Clearing
```typescript
// Track conversation ID from store
const activeConversationId = useChatStore((s) => s.activeConversationId);

// collapseTimerRef is defined in 13.6.1
// const collapseTimerRef = useRef<NodeJS.Timeout | null>(null);

// Clear timer and reset on conversation switch
useEffect(() => {
  // Clear any pending collapse timer when conversation changes
  if (collapseTimerRef.current) {
    clearTimeout(collapseTimerRef.current);
    collapseTimerRef.current = null;
  }
  // Reset to expanded on conversation switch
  setIsStepperExpanded(true);
}, [activeConversationId]);
```

### Timer Hygiene
This effect clears collapse timers on:
1. **Unmount** - useEffect cleanup (from 13.6.1)
2. **Conversation switch** - explicit clear via ref (this story)
3. **State rollback to error** - dependency change triggers 13.6.1 cleanup

### Why This Matters
Without timer clearing on conversation switch:
- User in Conversation A, generation completes
- Timer starts (800ms to collapse)
- User switches to Conversation B at 400ms
- At 800ms, timer fires and collapses B's stepper incorrectly

### Natural Reset via Component Lifecycle
Note: If QuestionnairePromptCard unmounts/remounts on conversation switch (due to different message list), the local state naturally resets. The explicit timer clearing is a safety guard for cases where the component persists.
