# Story 13.1 – Inline Questionnaire Bubble Integration

## Status: Not Started

## Overview
Move the questionnaire readiness/generation UI fully into the chat stream as an assistant-style bubble. Remove the detached container under the composer and rely on the inline bubble for actions/visibility. Do not build the progress stepper yet (that's Story 13.4); this story is layout/placement/state wiring only.

## Requirements
- Render the questionnaire prompt card inline within the chat message list (assistant-style bubble)
- Keep existing actions: Generate, Download (when available), Retry (for errors)
- Remove the old Dismiss control
- Preserve current readiness states (ready/generating/download/error)
- Layout/UX:
  - Bubble should align with assistant messages (max width similar to chat bubbles)
  - Consistent padding and rounded corners
  - **No separate avatar** - card appears inline after Guardian message which already has avatar

## Non-Goals (Deferred)
- Progress stepper or generation-phase indicators (Story 13.4/13.5)
- Durable download state/persistence changes (Story 13.3)
- Removing sticky indicator (Story 13.7)
- Removing "Guardian is thinking" banner (Story 13.8)
