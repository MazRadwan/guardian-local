# Story 13.1.3 - Update ChatInterface Dismiss Logic

## Status: Not Started

## Context
The `ChatInterface` component currently passes an `onDismiss` handler to `QuestionnairePromptCard`. Since Story 13.1.2 removed the dismiss button from the card, we need to remove the handler from ChatInterface as well.

## Goal
Remove dismiss-related handlers from ChatInterface while maintaining backwards compatibility with already-dismissed conversations.

## File to Modify
`apps/web/src/components/chat/ChatInterface.tsx`

## Current Implementation
The component currently has:

### handleDismiss Function
```tsx
const handleDismiss = useCallback(() => {
  if (!activeConversationId || !pendingQuestionnaire) return;

  // Mark as dismissed in localStorage
  dismiss(activeConversationId);

  // Clear from store
  clearPendingQuestionnaire();

  // Transition to hidden
  setQuestionnaireUIState('hidden');
}, [activeConversationId, pendingQuestionnaire, dismiss, clearPendingQuestionnaire, setQuestionnaireUIState]);
```

### QuestionnairePromptCard Usage
```tsx
<QuestionnairePromptCard
  ref={questionnaireCardRef}
  payload={pendingQuestionnaire}
  uiState={questionnaireUIState}
  error={questionnaireError}
  exportData={exportData}
  onGenerate={handleGenerateQuestionnaire}
  onDismiss={handleDismiss}  // <-- Remove this
  onDownload={handleDownload}
  onRetry={handleGenerateQuestionnaire}
  className="mx-4 mb-4"
/>
```

## Required Changes

### 1. Remove handleDismiss Function
Delete the entire `handleDismiss` callback function.

### 2. Remove onDismiss Prop from QuestionnairePromptCard
```tsx
<QuestionnairePromptCard
  ref={questionnaireCardRef}
  payload={pendingQuestionnaire}
  uiState={questionnaireUIState}
  error={questionnaireError}
  exportData={exportData}
  onGenerate={handleGenerateQuestionnaire}
  // onDismiss={handleDismiss}  <-- REMOVED
  onDownload={handleDownload}
  onRetry={handleGenerateQuestionnaire}
  className="mx-4 mb-4"
/>
```

### 3. Keep isDismissed Check (Backwards Compatibility)
**IMPORTANT**: Do NOT remove the `isDismissed` check in the rehydration useEffect. Users who previously dismissed a questionnaire should still have it hidden.

The rehydration logic should still include:
```tsx
// Skip if dismissed
if (isDismissed(activeConversationId)) {
  return;
}
```

### 4. Keep dismiss Function Import (For Now)
The `dismiss` function from `useQuestionnairePersistence` is still needed for:
- The `isDismissed` check in rehydration
- Will be fully removed in Story 13.7

Do NOT remove:
```tsx
const { dismiss, isDismissed, savePayload, loadPayload } = useQuestionnairePersistence(user?.id);
```

## Acceptance Criteria
- [ ] `handleDismiss` function removed from ChatInterface
- [ ] `onDismiss` prop removed from QuestionnairePromptCard usage
- [ ] `isDismissed` check still works in rehydration (backwards compat)
- [ ] `dismiss` function still imported (used by isDismissed)
- [ ] No runtime errors when viewing questionnaire card
- [ ] TypeScript compiles without errors
