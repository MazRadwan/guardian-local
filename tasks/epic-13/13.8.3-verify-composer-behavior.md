# Story 13.8.3 - Verify Composer Behavior

## Status: Not Started

## Context
After removing the thinking banner, verify that all other Composer behaviors remain intact.

## Goal
Ensure send/stop/disabled behaviors are unchanged after banner removal.

## Verification Steps

### 1. Build Check
```bash
pnpm --filter @guardian/web build
```

### 2. Test Suite
```bash
pnpm --filter @guardian/web test -- Composer
```

### 3. Manual Verification
```bash
pnpm --filter @guardian/web dev
```

## Behavior Checklist

### Must Still Work
- [ ] **Send button** - Disabled when loading or streaming
- [ ] **Textarea** - Disabled when loading or streaming
- [ ] **Stop button** - Appears during streaming
- [ ] **Mode selector** - Hidden during busy state
- [ ] **File upload** - Hidden during busy state
- [ ] **Keyboard shortcuts** - Enter to send, Shift+Enter for newline

### Must Be Gone
- [ ] **"Guardian is thinking" banner** - No longer appears during pre-stream loading

### Must Be Unchanged (Different Components)
- [ ] **MessageList typing indicator** - Still shows during streaming (`data-testid="typing-indicator"`)
- [ ] **Chat skeleton loaders** - Still show when loading empty conversation

## What NOT to Break

| Component | Location | Purpose |
|-----------|----------|---------|
| Typing indicator | MessageList | Shows during streaming |
| Skeleton loaders | MessageList | Shows while loading history |
| Stop button | Composer | Aborts streaming |
| Send button | Composer | Submits message |

## Acceptance Criteria
- [ ] `pnpm build` passes
- [ ] All Composer tests pass
- [ ] Send button works correctly
- [ ] Stop button works during streaming
- [ ] Textarea enables/disables correctly
- [ ] Mode selector shows/hides correctly
- [ ] No "Guardian is thinking" banner
- [ ] MessageList typing indicator still works
- [ ] No console errors
