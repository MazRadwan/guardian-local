# Story 13.2.2 - Add Export State Check to Rehydration

## Status: Not Started

## Context
Story 13.2.1 removed the dismiss check from rehydration. Now we need to add export state checking so that conversations with completed exports restore to 'download' state instead of regressing to 'ready'.

The `exportReadyByConversation` in chatStore is an in-memory cache that survives conversation switches but not page reloads. This story adds priority-based rehydration: export_ready → 'download', else payload → 'ready'.

## Goal
Update rehydration effect to check `exportReadyByConversation` FIRST, restoring to 'download' state if export exists for the conversation.

## File to Modify
`apps/web/src/components/chat/ChatInterface.tsx`

## Required Changes

### 1. Add Export State Check Before Payload Check
Insert priority check for export state:

**New Rehydration Logic:**
```typescript
useEffect(() => {
  if (prevConversationIdRef.current === activeConversationId) return;
  prevConversationIdRef.current = activeConversationId;
  if (isGeneratingQuestionnaire) return;

  // Clear previous conversation's UI state
  clearPendingQuestionnaire();
  setQuestionnaireUIState('hidden');

  // Priority 1: Check in-memory export state (survives conversation switch)
  const exportData = useChatStore.getState().exportReadyByConversation[activeConversationId];
  if (exportData && exportData.conversationId === activeConversationId) {
    // Restore to download state - export already completed for this conversation
    setQuestionnaireUIState('download');
    return;
  }

  // Priority 2: Check localStorage payload with shape validation
  const savedPayload = persistence.loadPayload(activeConversationId);
  if (savedPayload) {
    // Validate payload shape and conversation match
    if (savedPayload.conversationId === activeConversationId && savedPayload.assessmentType) {
      setPendingQuestionnaire(savedPayload);
      setQuestionnaireUIState('ready');
    } else {
      // Malformed or mismatched payload - clear to prevent state poisoning
      persistence.clearPayload(activeConversationId);
    }
  }
}, [activeConversationId, user?.id]);
```

### 3. No New Selector Needed
Use `useChatStore.getState()` directly in the effect since:
- This is a one-time check during rehydration, not reactive
- Avoids adding to dependency array
- Matches existing pattern in useWebSocketEvents

## State Priority Order

| Priority | Source | Result State | When |
|----------|--------|--------------|------|
| 1 | `exportReadyByConversation[conversationId]` | 'download' | Export completed, user switched away and back |
| 2 | `localStorage` payload via `persistence.loadPayload()` | 'ready' | questionnaire_ready received, no export yet |
| 3 | Neither | 'hidden' | No questionnaire activity for this conversation |

## Acceptance Criteria
- [ ] Rehydration checks export state FIRST
- [ ] If export exists → UI state 'download'
- [ ] If no export but payload exists → UI state 'ready', pendingQuestionnaire set
- [ ] If neither → UI state 'hidden'
- [ ] **Conversation match validated** - exportData.conversationId === activeConversationId
- [ ] **Payload shape validated** - savedPayload.conversationId matches AND assessmentType exists
- [ ] **Malformed data cleared** - poisoned entries removed via persistence.clearPayload()
- [ ] No regression in existing behavior
- [ ] No TypeScript errors
