# Story 13.9.5 - Observability and Logging

## Status: Not Started

## Context
The export status feature needs proper logging for debugging and monitoring. This story verifies logging is in place across backend and frontend.

## Goal
Ensure concise, useful logging for export status requests/responses without exposing sensitive data.

## Logging Strategy Note

**For MVP:** `console.log` / `console.error` is acceptable.

**Future Enhancement:** Prefer a structured logger (e.g., winston, pino) with:
- Log levels (debug, info, warn, error)
- Structured JSON output
- Request correlation IDs
- Easy swap-out via wrapper interface

If using `console` for MVP, consider wrapping via a logger interface for future swap:
```typescript
// lib/logger.ts
export const logger = {
  info: (...args: unknown[]) => console.log(...args),
  warn: (...args: unknown[]) => console.warn(...args),
  error: (...args: unknown[]) => console.error(...args),
};
```

## Verification Checklist

### Backend Logging (`packages/backend/src/infrastructure/websocket/ChatServer.ts`)

Verify these log statements exist in `handleGetExportStatus`:

```typescript
// On request received:
console.log(`[ChatServer] get_export_status request: conversationId=${conversationId}, userId=${userId}`);

// On export found:
console.log(`[ChatServer] export_status found: assessmentId=${assessment.id}, questions=${questionCount}`);

// On not found (assessment missing or no questions):
console.log(`[ChatServer] export_status not found: conversationId=${conversationId}`);

// On auth error (conversation not found or unauthorized):
console.log(`[ChatServer] export_status auth error: conversationId=${conversationId}, reason=${reason}`);

// On exception:
console.error(`[ChatServer] get_export_status error:`, error);
```

**Note:** Each path through `handleGetExportStatus` MUST log something. The "not found" log is required when:
- Assessment doesn't exist for conversation
- Assessment exists but has 0 questions

### Frontend Logging (`apps/web/src/lib/websocket.ts`)

Verify these log statements exist:

```typescript
// On request sent:
console.log('[WebSocket] Requesting export status for:', conversationId);

// On not found received:
console.log('[WebSocket] Export status not found:', data.conversationId);

// On error received:
console.error('[WebSocket] Export status error:', data);
```

### Frontend Logging (`apps/web/src/hooks/useWebSocketEvents.ts`)

Verify these log statements exist:

```typescript
// On cache hit (memory):
console.log('[useWebSocketEvents] Export data cached in memory, skipping server request');

// On cache hit (localStorage):
console.log('[useWebSocketEvents] Export data found in localStorage, restoring');

// On server request:
console.log('[useWebSocketEvents] Requesting export status from server');

// On not found response:
console.log('[useWebSocketEvents] No export found for conversation, user can generate');

// On error response:
console.error('[useWebSocketEvents] Export status error:', data.error);
```

## Security Checklist

Ensure NO logs contain:
- [ ] User passwords or tokens
- [ ] Full conversation content
- [ ] Personal identifiable information (PII)
- [ ] API keys or secrets

Logs SHOULD contain:
- [ ] Conversation IDs (for tracing)
- [ ] Assessment IDs (for correlation)
- [ ] User IDs (for ownership context)
- [ ] Error messages (for debugging)
- [ ] Question counts (for verification)

## Log Output Examples

### Happy Path (Export Found)
```
[ChatServer] get_export_status request: conversationId=conv-123, userId=user-456
[ChatServer] export_status found: assessmentId=assess-789, questions=42
```

### Cache Hit Path
```
[useWebSocketEvents] Export data cached in memory, skipping server request
```

### Not Found Path
```
[ChatServer] get_export_status request: conversationId=conv-123, userId=user-456
[ChatServer] export_status not found: conversationId=conv-123
[useWebSocketEvents] No export found for conversation, user can generate
```

### Error Path
```
[ChatServer] get_export_status request: conversationId=conv-123, userId=user-456
[ChatServer] export_status auth error: conversationId=conv-123, reason=Unauthorized
[useWebSocketEvents] Export status error: Unauthorized
```

## Acceptance Criteria
- [ ] Backend logs request received with conversationId and userId
- [ ] Backend logs export found with assessmentId and questionCount
- [ ] Backend logs not found with conversationId
- [ ] Backend logs auth errors with reason
- [ ] Backend logs exceptions with error object
- [ ] Frontend logs request sent
- [ ] Frontend logs cache hits (memory and localStorage)
- [ ] Frontend logs not found response
- [ ] Frontend logs error response
- [ ] No sensitive data in any logs
- [ ] Build passes
