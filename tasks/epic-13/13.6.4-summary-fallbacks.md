# Story 13.6.4 - Add Safe Fallbacks for Summary Data

## Status: Complete

## Context
The collapsed summary shows question count and assessment type. If this data is missing, the UI should gracefully degrade rather than break.

## Goal
Handle missing `questionCount` and `assessmentType` in the collapsed summary with safe fallbacks.

## File to Modify
`apps/web/src/components/chat/QuestionnairePromptCard.tsx`

## Implementation

### Current Code (potentially unsafe)
```typescript
{isComplete && !isStepperExpanded && (
  <span className="text-xs text-slate-500">
    {payload.questionCount} questions · {payload.assessmentType}
  </span>
)}
```

### Updated Code with Fallbacks
```typescript
{isComplete && !isStepperExpanded && (
  <div className="px-3 pb-2.5 -mt-1">
    <span className="text-xs text-slate-500">
      {payload?.questionCount
        ? `${payload.questionCount} questions`
        : 'Assessment complete'}
      {payload?.assessmentType && ` · ${payload.assessmentType}`}
    </span>
  </div>
)}
```

### Fallback Strategy

| Scenario | questionCount | assessmentType | Display |
|----------|---------------|----------------|---------|
| Both present | 40 | "Quick Assessment" | "40 questions · Quick Assessment" |
| Only count | 40 | undefined | "40 questions" |
| Only type | undefined | "Quick Assessment" | "Assessment complete · Quick Assessment" |
| Neither | undefined | undefined | "Assessment complete" |

### Alternative Approach (more defensive)
```typescript
const getSummaryText = () => {
  const parts: string[] = [];

  if (payload?.questionCount) {
    parts.push(`${payload.questionCount} questions`);
  } else {
    parts.push('Assessment complete');
  }

  if (payload?.assessmentType) {
    parts.push(payload.assessmentType);
  }

  return parts.join(' · ');
};

// In JSX:
{isComplete && !isStepperExpanded && (
  <div className="px-3 pb-2.5 -mt-1">
    <span className="text-xs text-slate-500">
      {getSummaryText()}
    </span>
  </div>
)}
```

## Acceptance Criteria

- [ ] Shows "Assessment complete · Type" when only type present
- [ ] Shows "Assessment complete" when neither present
