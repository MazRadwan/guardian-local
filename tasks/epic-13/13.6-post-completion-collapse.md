**Story 13.6 – Post-Completion Collapse & Download Presentation**

Goal: Refine the post-completion UX for the questionnaire bubble/stepper: auto-collapse after completion, preserve a concise summary when collapsed, and present download buttons cleanly within the collapsed/expandable UI. This builds on 13.4’s stepper scaffold and 13.5’s real phase wiring.

Scope:
- Frontend only (apps/web).
- Likely touch:
  - `apps/web/src/components/chat/QuestionnairePromptCard.tsx` (collapse logic, summary line, download layout)
  - `apps/web/src/components/chat/ChatInterface.tsx` (prop/state plumbing if needed)
  - Any stepper-related component/hook introduced in 13.4 (to add completion handling)
  - Tests under `apps/web/src/components/chat/__tests__/` for collapsed state and downloads.

Requirements:
- When the stepper reaches “complete”, auto-collapse after a short delay (similar to the demo behavior), but allow the user to re-expand.
- In collapsed state, show a concise summary line (e.g., question count/assessment type or payload summary) and keep download buttons accessible.
- Ensure download buttons remain visible/usable both when collapsed and when re-expanded; no state clearing on download (respect 13.3 durability).
- Maintain existing state transitions and phase events from 13.5; do not alter backend wiring.
- Keep selectors/testids stable where possible; add only if needed for collapsed summary or download controls.

Non-goals:
- Changing download durability semantics (already in 13.3).
- Removing sticky indicator or “Guardian is thinking” banner.
- Redefining phase events (covered by 13.5).

Testing:
- Add/adjust component tests to verify:
  - Auto-collapse occurs after completion.
  - Summary text and download buttons appear and remain functional when collapsed.
  - Manual expand/collapse works after completion.
- Keep e2e selectors stable if possible; adjust only if the post-completion layout changes visibility/placement.

Deliverables:
- Post-completion UX that auto-collapses with a readable summary and persistent download buttons.
- Tests updated to cover collapsed/expanded post-completion states.

### Summary of Sub-Stories

1. **13.6.1 - Completion Guards**: Only auto-collapse on successful completion (`uiState === 'download'`), not on error. Keep stepper expanded on error to show failure point. Uses timer ref pattern for proper cleanup.

2. **13.6.2 - Conversation Scoping**: Reset collapse state when conversation changes. Clear pending collapse timers on conversation switch. New conversations start fresh (expanded when generating).

3. **13.6.3 - Download Buttons Layout**: Position download buttons OUTSIDE the collapsible section. Visible and functional in both collapsed and expanded states. Honors 13.3 durability (no state clearing on download).

4. **13.6.4 - Summary Fallbacks**: Handle missing `questionCount` and `assessmentType` gracefully. Shows "Assessment complete" when data missing, no orphaned separators.

5. **13.6.5 - Component Tests**: Comprehensive tests using fake timers for:
   - Auto-collapse timing (800ms after success)
   - No collapse on error or complete→error transition
   - Conversation scoping (no cross-convo bleed)
   - Timer cleared on conversation switch mid-delay (NEW)
   - Download visibility and functionality when collapsed
   - Download durability and multiple downloads
   - Summary fallbacks for all data combinations
   - Use `GENERATION_STEPS.length` instead of hardcoded step counts
