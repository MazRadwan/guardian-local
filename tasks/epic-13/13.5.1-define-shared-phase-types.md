# Story 13.5.1 - Define Shared Phase Types

## Status: Not Started

## Context
Story 13.5 wires the stepper to real backend phase events. This sub-story creates the shared types used by both backend (emitting) and frontend (listening).

## Goal
Create shared TypeScript types for generation phase events that can be imported by both packages.

## File to Modify
`packages/shared/src/types/index.ts`

## Implementation

### Add Phase Types
```typescript
// Generation phase event payload
export interface GenerationPhasePayload {
  conversationId: string;
  phase: number;
  phaseId: 'context' | 'generating' | 'validating' | 'saving';
  timestamp: number;
}

// Phase definitions (matches GENERATION_STEPS in chatStore)
export const GENERATION_PHASES = ['context', 'generating', 'validating', 'saving'] as const;
export type GenerationPhaseId = typeof GENERATION_PHASES[number];
```

### Phase Mapping
| Phase Index | Phase ID | Step Label | Description |
|-------------|----------|------------|-------------|
| 0 | context | Context gathered | Input validated, prompt built |
| 1 | generating | Generating questions | Claude API call completed |
| 2 | validating | Validating structure | Schema extracted and validated |
| 3 | saving | Saving assessment | Questions persisted to database |
| 4 | (complete) | N/A | Signaled by `export_ready` event |

## Acceptance Criteria
- [ ] `GenerationPhasePayload` interface exported from shared types
- [ ] `GENERATION_PHASES` constant exported (readonly tuple)
- [ ] `GenerationPhaseId` type exported
- [ ] Types importable by backend: `import { GenerationPhasePayload } from '@guardian/shared'`
- [ ] Types importable by frontend: `import { GenerationPhasePayload } from '@guardian/shared'`
- [ ] No runtime dependencies (types only, except for const array)
- [ ] Build passes for both packages
