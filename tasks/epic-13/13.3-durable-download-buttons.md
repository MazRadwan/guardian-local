# Story 13.3 – Durable Download Buttons

## Status: Not Started

## Overview
Make questionnaire downloads idempotent and persistent. After download, keep the download state visible so users can re-download without regenerating.

## Requirements
- After successful download, do NOT clear pendingQuestionnaire/payload or hide the card
- Keep download state visible so users can re-download
- Download action fetches `/api/assessments/:assessmentId/export/:format` using stored `assessmentId`
- No state reset on success; on error, keep state for retry
- Keep existing readiness/error transitions intact (ready → generating → download/error)
- Remove only the post-download "clear and hide" behavior
- Persist export state to localStorage for page reload survival

## Security & Validation Guards

| Guard | Location | Purpose |
|-------|----------|---------|
| **Conversation match** | loadExport | Verify export.conversationId === requested conversationId |
| **Shape validation** | loadExport | Check required fields: assessmentId, formats array |
| **Malformed cleanup** | loadExport | Auto-clear poisoned localStorage entries |
| **User-scoped keys** | saveExport/loadExport | Keys include userId to prevent cross-user leaks |
| **Fresh event wins** | handleExportReady | export_ready overwrites both store AND localStorage |
| **Active-convo gating** | rehydration effect | Only restore for activeConversationId |
