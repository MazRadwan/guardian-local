# Story 13.2.1 - Remove Dismiss Check from Rehydration

## Status: Not Started

## Context
Story 13.1 removed the dismiss button from QuestionnairePromptCard. Story 13.2 removes the dismiss-related logic from rehydration to ensure questionnaire UI state is always restored when switching conversations.

## Goal
Remove the `isDismissed()` check from the rehydration effect so that dismissals no longer block restoration of questionnaire state.

## File to Modify
`apps/web/src/components/chat/ChatInterface.tsx`

## Current Implementation
The rehydration effect (lines ~66-98) currently checks dismiss flag before restoring:

```typescript
useEffect(() => {
  if (prevConversationIdRef.current === activeConversationId) return;
  prevConversationIdRef.current = activeConversationId;
  if (isGeneratingQuestionnaire) return;

  // Clear previous conversation's state
  clearPendingQuestionnaire();
  setQuestionnaireUIState('hidden');

  // Respect dismiss flag - don't restore if dismissed  <-- REMOVE THIS CHECK
  if (persistence.isDismissed(activeConversationId)) return;

  // Try to load persisted payload
  const savedPayload = persistence.loadPayload(activeConversationId);
  if (savedPayload) {
    setPendingQuestionnaire(savedPayload);
    setQuestionnaireUIState('ready');
  }
}, [activeConversationId, user?.id]);
```

## Required Changes

### 1. Remove isDismissed Check
Remove the dismiss flag check from the rehydration effect:

**Before:**
```typescript
// Respect dismiss flag - don't restore if dismissed
if (persistence.isDismissed(activeConversationId)) return;

// Try to load persisted payload
const savedPayload = persistence.loadPayload(activeConversationId);
```

**After:**
```typescript
// Try to load persisted payload (dismiss flag no longer checked)
const savedPayload = persistence.loadPayload(activeConversationId);
```

### 2. Remove Any Remaining persistence.dismiss() Calls
If `handleDismiss` still exists and calls `persistence.dismiss()`, remove that call:

**Before (if exists):**
```typescript
const handleDismiss = useCallback(() => {
  persistence.dismiss(activeConversationId);  // <-- REMOVE
  clearPendingQuestionnaire();
  setQuestionnaireUIState('hidden');
}, [activeConversationId, persistence]);
```

**After:**
```typescript
const handleDismiss = useCallback(() => {
  clearPendingQuestionnaire();
  setQuestionnaireUIState('hidden');
}, []);
```

Note: If Story 13.1.3 already removed `handleDismiss` entirely, this step is N/A.

### 3. Keep persistence Hook for Payload Operations
Do NOT remove the `useQuestionnairePersistence` hook - it's still needed for:
- `savePayload()` - called by `handleQuestionnaireReady` in useWebSocketEvents
- `loadPayload()` - called in rehydration
- `clearPayload()` - called after download

## Acceptance Criteria
- [ ] `isDismissed()` check removed from rehydration effect
- [ ] `persistence.dismiss()` calls removed (if any remain)
- [ ] `persistence` hook still imported and used for payload operations
- [ ] No TypeScript errors
- [ ] Rehydration restores payload regardless of previous dismiss state
