# Story 13.6.3 - Ensure Download Buttons in Both States

## Status: Not Started

## Context
Download buttons should be visible and functional whether the stepper is collapsed or expanded. They should respect 13.3 durability (no state clearing on download).

## Goal
Ensure download buttons are always visible when generation is complete, regardless of collapse state.

## File to Modify
`apps/web/src/components/chat/QuestionnairePromptCard.tsx`

## Implementation

### Key Principle
Download buttons should be OUTSIDE the collapsible section, not inside it.

### Layout Structure
```typescript
{/* Stepper section with collapse */}
{(isGenerating || isComplete) && (
  <div className="border rounded-lg bg-white overflow-hidden">
    {/* Clickable header - always visible */}
    <button onClick={toggleExpand}>
      {/* Header content */}
    </button>

    {/* Collapsible stepper content */}
    <div className={`transition-all ${isStepperExpanded ? 'max-h-64' : 'max-h-0'}`}>
      <VerticalStepper ... />
    </div>

    {/* Summary when collapsed + complete - inside card border */}
    {isComplete && !isStepperExpanded && (
      <div className="px-3 pb-2.5">
        <span className="text-xs text-slate-500">
          {questionCount} questions · {assessmentType}
        </span>
      </div>
    )}
  </div>
)}

{/* Download buttons - OUTSIDE the collapsible card, always visible when complete */}
{isComplete && uiState === 'download' && (
  <div className="flex items-center gap-2 mt-3 animate-fadeIn">
    <button
      onClick={() => onDownload('word')}
      className="inline-flex items-center gap-1.5 px-4 py-2 bg-slate-800 text-white rounded-lg"
    >
      Word
    </button>
    <button
      onClick={() => onDownload('pdf')}
      className="px-3 py-2 text-sm text-slate-600 hover:bg-slate-100 rounded-lg"
    >
      PDF
    </button>
    <button
      onClick={() => onDownload('excel')}
      className="px-3 py-2 text-sm text-slate-600 hover:bg-slate-100 rounded-lg"
    >
      Excel
    </button>
  </div>
)}
```

### Download Handler (from parent)
The `onDownload` handler should:
1. Use `exportReadyByConversation[conversationId]` data
2. NOT clear export state after download
3. Support multiple downloads

```typescript
// In ChatInterface or parent:
const handleDownload = useCallback((format: 'word' | 'pdf' | 'excel') => {
  const exportData = exportReadyByConversation[activeConversationId];
  if (!exportData) return;

  // Download using assessmentId from exportData
  downloadQuestionnaire(exportData.assessmentId, format);
  // Note: Do NOT clear exportReadyByConversation here (13.3 durability)
}, [activeConversationId, exportReadyByConversation]);
```
