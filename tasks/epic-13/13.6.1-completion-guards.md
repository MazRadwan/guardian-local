# Story 13.6.1 - Add Completion Guards for Auto-Collapse

## Status: Complete

## Context
Story 13.4.4 added auto-collapse behavior after completion. This story adds guards to only collapse on successful completion, not on error. Uses ref for timer hygiene to enable 13.6.2's conversation-switch clearing.

## Goal
Only auto-collapse when generation completes successfully (`uiState === 'download'`), not when an error occurs. Use timer ref pattern for proper cleanup.

## File to Modify
`apps/web/src/components/chat/QuestionnairePromptCard.tsx`

## Implementation

### Current Code (from 13.4.4)
```typescript
useEffect(() => {
  if (isComplete) {
    const timer = setTimeout(() => setIsStepperExpanded(false), 800);
    return () => clearTimeout(timer);
  }
}, [isComplete]);
```

### Updated Code with Guards and Timer Ref
```typescript
import { useRef } from 'react';

// Add ref for timer (enables manual clearing in 13.6.2)
const collapseTimerRef = useRef<NodeJS.Timeout | null>(null);

// Guard: only collapse on successful completion
const isSuccessfullyComplete = isComplete && uiState === 'download';

useEffect(() => {
  // Only auto-collapse on successful completion, not error
  if (isSuccessfullyComplete) {
    collapseTimerRef.current = setTimeout(() => {
      setIsStepperExpanded(false);
      collapseTimerRef.current = null;
    }, 800);
    return () => {
      if (collapseTimerRef.current) {
        clearTimeout(collapseTimerRef.current);
        collapseTimerRef.current = null;
      }
    };
  }
}, [isSuccessfullyComplete]);

// Keep expanded on error to show where it failed
useEffect(() => {
  if (uiState === 'error') {
    setIsStepperExpanded(true);
  }
}, [uiState]);
```

### Timer Ref Pattern
The `collapseTimerRef` pattern enables:
1. **Unmount cleanup**: useEffect cleanup clears timer
2. **Conversation switch**: 13.6.2 can clear timer via ref
3. **State rollback**: Dependency change triggers cleanup

### Why This Guard Matters
| Scenario | Old Behavior | New Behavior |
|----------|--------------|--------------|
| Successful completion | Collapse after 800ms | Collapse after 800ms ✓ |
| Error during generation | Would still collapse | Stay expanded to show failure point |
| Complete then error | Would collapse | Timer cleared, stay expanded |

## Acceptance Criteria

- [ ] Timer ref nullified after callback executes
- [ ] Error state (`uiState === 'error'`) keeps stepper expanded
