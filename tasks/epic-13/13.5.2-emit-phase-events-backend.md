# Story 13.5.2 - Emit Phase Events from Backend

## Status: Complete

## Context
The shared types are defined (13.5.1). This story adds phase event emissions to the backend ChatServer during questionnaire generation.

## Goal
Emit `generation_phase` events at each phase boundary in the `handleGenerateQuestionnaire` flow.

## File to Modify
`packages/backend/src/infrastructure/websocket/ChatServer.ts`

## Implementation

### Add Import
```typescript
import { GenerationPhasePayload, GENERATION_PHASES } from '@guardian/shared';
```

### Add Helper Method
```typescript
/**
 * Emit a generation phase event to the client
 */
private emitGenerationPhase(
  socket: Socket,
  conversationId: string,
  phase: number,
  phaseId: string
): void {
  const payload: GenerationPhasePayload = {
    conversationId,
    phase,
    phaseId: phaseId as GenerationPhasePayload['phaseId'],
    timestamp: Date.now(),
  };
  socket.emit('generation_phase', payload);
}
```

### Modify handleGenerateQuestionnaire
Insert phase emissions at the appropriate points:

```typescript
// In handleGenerateQuestionnaire():

// Phase 0: Context ready (after validation, before Claude call)
this.emitGenerationPhase(socket, conversationId, 0, 'context');

// Call Claude (longest phase - 10-15s)
const result = await this.questionnaireGenerationService.generate(context);

// Phase 1: Generating complete (Claude returned)
this.emitGenerationPhase(socket, conversationId, 1, 'generating');

// Validation happens inside generate(), but we emit after
// Phase 2: Validating complete
this.emitGenerationPhase(socket, conversationId, 2, 'validating');

// Persistence
await this.persistQuestionnaire(result);

// Phase 3: Saving complete
this.emitGenerationPhase(socket, conversationId, 3, 'saving');

// export_ready already signals completion (phase 4)
socket.emit('export_ready', { ... });
```

### Emission Points
| Line (approx) | Phase | Event |
|---------------|-------|-------|
| Before `generate()` call | 0 | `generation_phase { phase: 0, phaseId: 'context' }` |
| After `generate()` returns | 1 | `generation_phase { phase: 1, phaseId: 'generating' }` |
| After validation logic | 2 | `generation_phase { phase: 2, phaseId: 'validating' }` |
| After persistence | 3 | `generation_phase { phase: 3, phaseId: 'saving' }` |
| After markdown ready | 4 | `export_ready` (existing event) |

### Error Handling
- If error occurs after phase N, phases 0..N will have been emitted
- `extraction_failed` event still emitted on error (existing behavior)
- No cleanup of phase events needed (frontend resets on error)
