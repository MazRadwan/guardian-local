# Story 13.2.3 - Remove clearDismiss Call from Event Handler

## Status: Not Started

## Context
With dismiss functionality removed in Story 13.1 and dismiss checks removed in Story 13.2.1, the `clearDismiss()` call in `handleQuestionnaireReady` is now dead code. This story removes it.

## Goal
Remove the `persistence.clearDismiss()` call from the `handleQuestionnaireReady` event handler since dismiss flags are no longer used.

## File to Modify
`apps/web/src/hooks/useWebSocketEvents.ts`

## Current Implementation
The `handleQuestionnaireReady` function (lines ~389-423) calls both `clearDismiss` and `savePayload`:

```typescript
const handleQuestionnaireReady = useCallback((data: QuestionnaireReadyPayload) => {
  // ... guards and validation ...

  // Story 4.3.5: Clear dismiss flag and save to localStorage
  if (persistence) {
    persistence.clearDismiss(data.conversationId);  // <-- REMOVE
    persistence.savePayload(data.conversationId, data);
  }

  // Update store
  useChatStore.getState().setPendingQuestionnaire(data);
  useChatStore.getState().setQuestionnaireUIState('ready');
  useChatStore.getState().setQuestionnaireError(null);
}, [persistence, activeConversationId]);
```

## Required Changes

### 1. Remove clearDismiss Call

**Before:**
```typescript
// Story 4.3.5: Clear dismiss flag and save to localStorage
if (persistence) {
  persistence.clearDismiss(data.conversationId);
  persistence.savePayload(data.conversationId, data);
}
```

**After:**
```typescript
// Save questionnaire payload to localStorage for rehydration
if (persistence) {
  persistence.savePayload(data.conversationId, data);
}
```

### 2. Update Comment
Update the comment to reflect current behavior (no longer referencing dismiss flag clearing).
