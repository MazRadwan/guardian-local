### Requirements
- Apply Appendix example styling (`tasks/epic-14/Appendix-example-code.md`) to generate and download buttons (color, padding, hover/focus) without altering chat message layout.
- Primary download button matches the reference; secondary buttons use light/ghost treatment.
- Preserve existing data-testid hooks and disabled/loading states.
- No new “special” bubble/card; styling is confined to buttons within the standard chat message.

### Design Decisions
| Decision | Choice | Rationale |
|----------|--------|-----------|
| Primary/secondary | First download button as primary; others secondary | Matches reference and current behavior |
| Layout | Keep buttons inline within chat message content | Prevents bespoke card regression |
| Reuse components | Prefer existing `Button` component with class overrides | Consistency and theme alignment |

### Testing
