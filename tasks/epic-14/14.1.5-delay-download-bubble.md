## Story 14.1.5 – Delay Download Bubble Until Questionnaire Renders

Status: Complete

### Goal
Ensure the download bubble/card appears immediately after the streamed questionnaire message (not before it), so the download buttons stay in view when the questionnaire finishes rendering.

### Context
- Current flow: `questionnaire_ready` sets `pendingQuestionnaire` and captures `questionnaireMessageIndex` (messages length at that moment). `export_ready` flips UI to `download`, so the download bubble can render above the streamed questionnaire content.
- We already track `questionnaireMessageIndex` and have `handleStreamComplete` (called when message streaming ends).

### Approach (low refactor)
- On stream completion (or when questionnaire message mounts), update `questionnaireMessageIndex` to the end of the list so the questionnaire/download bubble renders below the streamed questionnaire content.
- Optionally gate download visibility: cache `export_ready` data, but only show the download UI after stream completion/mount of the questionnaire bubble.
- Keep generation/download coupling; no backend or schema changes.
- Auto-scroll the newly shown download bubble into view when it mounts.

### Acceptance Criteria
- When export_ready arrives before the questionnaire finishes streaming, the download bubble waits and renders below the questionnaire once it is present.
- Stepper bubble remains in its original position; download bubble appears after the questionnaire bubble.
- Rehydration retains the correct ordering (rehydrated conversations still place download below questionnaire).
- No regression to existing generation/error/download flows.

### Files & Expected Changes
- `apps/web/src/hooks/useWebSocketEvents.ts` (use `handleStreamComplete` or similar hook to set insertion position / visibility)
- `apps/web/src/stores/chatStore.ts` (possible helper to update `questionnaireMessageIndex` after stream)
- `apps/web/src/components/chat/MessageList.tsx` / `QuestionnaireMessage.tsx` (optional visibility gate / scroll-into-view)
- `apps/web/src/components/chat/ChatInterface.tsx` (wire updated index/visibility)

### Testing
- Unit/component: assertions that download buttons are hidden until stream-complete signal, then render after the questionnaire bubble.
- Update existing button styling tests (14.2.3) to trigger the “stream complete” path before expecting download buttons.
- Integration: ordering of stepper → questionnaire message → download bubble; rehydration ordering.
