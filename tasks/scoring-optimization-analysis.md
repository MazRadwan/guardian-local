# Scoring Workflow Optimization Analysis

**Date:** 2026-02-16
**Context:** Post-Epic 38 ISO enrichment, scoring takes ~7 min (was ~3 min)
**Verified:** Scoring works end-to-end (SuperHealthyCan.ai scored 52/100, commit `0eb2526`)

---

| Step | Duration | % of Total | Claude Call? |

## Bottleneck #1: Document Extraction (~5 min) — CRITICAL — ELIMINABLE

**File:** `packages/backend/src/infrastructure/ai/DocumentParserService.ts` (784 LOC — exceeds 300 LOC limit)

### KEY INSIGHT: Questions are already in the DB — skip Claude entirely

The questionnaire was **generated by Guardian** and exported via `WordExporter.ts`. The document
format is deterministic:

```
Question 1.1
{question text}
Response:
{answer text}

Question 1.2
...
```

We already have all 45 questions in the `questions` table with exact section/question numbers.
Instead of sending 100K chars to Claude and asking it to parse questions + answers:

1. Extract raw text from docx (mammoth — already done, takes milliseconds)
2. Split on `Question X.Y` regex markers
3. For each section, grab everything after `Response:` up to the next `Question` marker
4. Match to DB questions by section_number + question_number

**Impact: Eliminates the entire 5-minute Claude extraction call. ~7 min → ~2 min.**

**Fallback:** Keep Claude extraction as fallback for non-Guardian documents (third-party uploads).

### Other extraction issues (lower priority if regex approach works)

| Issue | Impact | Fix |
|-------|--------|-----|
| `DEFAULT_MAX_EXTRACTED_TEXT_CHARS = 100000` (line 47) — sends 100K chars to Claude | ~25K input tokens | Reduce to 50-75K; questionnaires don't need 100K |
| Response truncation to 2000 chars happens AFTER extraction (line 435-437) | Wasted tokens — Claude extracts 5K+ per response, then we truncate | Move truncation instruction INTO the extraction prompt |
| No intermediate progress during 5-min Claude call | UX dead zone | Add streaming progress callback |
| Prompt caching enabled but system prompt is identical across all parses | Already good | Verify cache hit rate |

## Bottleneck #2: Scoring Call (~2 min)

**Files:**
- `packages/backend/src/application/services/ScoringLLMService.ts` (139 LOC)
- `packages/backend/src/infrastructure/ai/prompts/scoringPrompt.ts`
- `packages/backend/src/infrastructure/ai/prompts/scoringPrompt.iso.ts`

| Issue | Impact | Fix |
|-------|--------|-----|
| ISO catalog injected into system prompt → breaks prompt caching | Cache miss on every call | Split: static system prompt (cacheable) + ISO catalog as user-prompt appendix |
| ISO catalog fetched from DB on every scoring call (2 sequential queries) | ~100ms + no caching | Cache ISO catalog in-memory on service init |
| Tool JSON output is 54,626 chars (~13.6K tokens) | Long generation time | Consider flattening tool schema or reducing nested detail |
| `maxTokens: 16384` | Required — tool JSON alone is 54K chars | **Do NOT reduce** — 10K proved insufficient |

## Bottleneck #4: Progress Feedback Gaps

**Current UX:** User sees 3 generic messages for 7 minutes:
1. "Extracting responses from document..." (5 min)
2. "Analyzing scoring..." (2 min)
3. "Scoring complete!"

**Recommended intermediate events:**
- During extraction: "Extracting responses... processing section X of Y"
- During scoring: "Scoring dimension X of 10..." or stream partial narrative
- After extraction: "Found N responses across M sections"
- Before scoring: "Preparing risk assessment with ISO compliance analysis..."

## Priority Action Items

### P0 — High Impact
1. **REGEX EXTRACTION for Guardian questionnaires** — skip Claude extraction entirely for known-format docs. Questions are in DB, document format is deterministic. **Saves ~5 min (71% of total time).** Keep Claude as fallback for non-Guardian documents.
2. **Cache ISO catalog in-memory** — skip 2 DB queries per scoring (ISOControlRetrievalService.ts)
3. **Split ISO from system prompt** — enable prompt caching on static portion (scoringPrompt.ts)
4. **Add intermediate progress events** during Claude streaming calls

### P1 — Medium Impact
5. **Move response truncation to extraction prompt** — don't extract 5K then truncate to 2K
6. **Combine ISO DB queries into 1 JOIN** (DrizzleDimensionControlMappingRepository.ts)
7. **Make ISO injection conditional** — skip if assessment has no ISO-mapped dimensions

### P2 — Code Quality
8. **Split DocumentParserService.ts** (784 LOC → 3 focused modules)
9. **Split ClaudeClient.ts** (844 LOC → 3 interface-specific modules)
10. **Flatten tool schema** — reduce nesting depth in scoringComplete.ts
