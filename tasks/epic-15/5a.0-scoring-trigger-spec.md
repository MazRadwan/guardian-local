## Implementation Approach for Story 5a.4

### Step 1: Wire uploadMode Dynamically (Composer.tsx)

**Current (Line 77):**
```typescript
const uploadMode: UploadMode = 'intake';  // HARDCODED
```

**Fix:**
```typescript
const uploadMode: UploadMode = currentMode === 'scoring' ? 'scoring' : 'intake';
```

### Step 2: Add Validation Gates in ScoringService

Before calling Claude for scoring, add:

```typescript
// 1. Check assessment status >= 'exported'
if (!['exported', 'questions_generated', 'scored'].includes(assessment.status)) {
  throw new ScoringError('ASSESSMENT_NOT_EXPORTED', 'Assessment has not been exported to vendor yet');
}

// 2. Check parseConfidence >= 0.7
if (parseResult.confidence < 0.7) {
  throw new ScoringError('PARSE_CONFIDENCE_TOO_LOW', `Parse confidence ${parseResult.confidence} is below minimum 0.7`);
}

// 3. Check rate limit (5 per day per assessment)
const todayCount = await this.scoringResultRepo.countTodayForAssessment(assessmentId);
if (todayCount >= 5) {
  throw new ScoringError('RATE_LIMITED', 'Maximum 5 scoring attempts per day exceeded');
}

// 4. Check file hash not recently scored (1hr window)
const recentHash = await this.scoringResultRepo.findRecentByFileHash(fileHash, 1);
if (recentHash) {
  throw new ScoringError('DUPLICATE_FILE', 'This file was scored within the last hour');
}
```

### Step 3: Update WebSocket Events

**Add scoring_started event:**
```typescript
socket.emit('scoring_started', {
  assessmentId,
  fileId,
  parseConfidence
});
```

**Add error codes to scoring_error:**
```typescript
socket.emit('scoring_error', {
  error: humanReadableMessage,
  code: 'ASSESSMENT_NOT_FOUND' | 'UNAUTHORIZED_ASSESSMENT' | 'ASSESSMENT_NOT_EXPORTED' |
        'PARSE_FAILED' | 'PARSE_CONFIDENCE_TOO_LOW' | 'RATE_LIMITED' | 'DUPLICATE_FILE' | 'SCORING_FAILED'
});
```

## WebSocket Event Contract

### Established Events (Already Working)
- `upload_progress`: {progress, stage, message, error?}
- `intake_context_ready`: {success, context, error?}
- `scoring_parse_ready`: {success, assessmentId, responses[], confidence, error?}

### New Events (Need Implementation in 5a.4)

```typescript
interface ScoringStartedEvent {
  event: 'scoring_started';
  assessmentId: string;
  fileId: string;
  parseConfidence: number;
}

interface ScoringProgressEvent {
  event: 'scoring_progress';
  status: 'parsing' | 'validating' | 'scoring' | 'storing';
  message: string;
  progress: number; // 0-100
}

interface ScoringCompleteEvent {
  event: 'scoring_complete';
  assessmentId: string;
  batchId: string;
  result: ScoringResultSummary;
}

interface ScoringErrorEvent {
  event: 'scoring_error';
  error: string;
  code: ScoringErrorCode;
}

type ScoringErrorCode =
  | 'ASSESSMENT_NOT_FOUND'
  | 'UNAUTHORIZED_ASSESSMENT'
  | 'ASSESSMENT_NOT_EXPORTED'
  | 'PARSE_FAILED'
  | 'PARSE_CONFIDENCE_TOO_LOW'
  | 'RATE_LIMITED'
  | 'DUPLICATE_FILE'
  | 'SCORING_FAILED';
```
