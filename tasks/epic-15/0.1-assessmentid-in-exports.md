# Story 0.1: AssessmentId in Questionnaire Exports

## Overview

Add assessment ID to all questionnaire exports (PDF, Word, Excel) so scoring can link responses back to the original assessment. This is the prerequisite for the entire scoring workflow.

**Phase:** 0 (Prerequisite) | **Parallel:** No | **Agent:** `chat-backend-agent`

---

- **Depends on:** None (first story)
- **Blocks:** All Epic 15 stories (1.1 through 6.1)

## Upstream Verification

> **IMPORTANT:** Before implementing export changes, verify that the upstream questionnaire generation
> service passes the assessmentId to the export process.

### Check QuestionnaireGenerationService

The service that generates questionnaires must include `assessmentId` in the data passed to exporters:

```typescript
// In QuestionnaireGenerationService or similar
async generateAndExport(assessmentId: string, ...): Promise<ExportResult> {
  const questionnaire = await this.generateQuestionnaire(assessmentId, ...);

  // VERIFY: assessmentId is included in export data
  const exportData: QuestionnaireExportData = {
    metadata: {
      assessmentId, // This MUST be present
      vendorName,
      solutionName,
      // ...
    },
    sections: questionnaire.sections,
  };

  return await this.exporter.export(exportData);
}
```

```typescript
describe('PDFExporter', () => {
  it('should include assessmentId in generated PDF', async () => {
    const testData = createTestQuestionnaireData({
      assessmentId: 'a1b2c3d4-5678-90ab-cdef-1234567890ab'

    const pdfBuffer = await exporter.generatePDF(testData);
    expect(pdfBuffer).toBeInstanceOf(Buffer);
    expect(pdfBuffer.length).toBeGreaterThan(0);
  });

  it('should escape HTML special characters in assessmentId', async () => {
    const testData = createTestQuestionnaireData({
      assessmentId: '<script>alert("xss")</script>'
    });
    const pdfBuffer = await exporter.generatePDF(testData);
    expect(pdfBuffer).toBeInstanceOf(Buffer);
  });
});
```
