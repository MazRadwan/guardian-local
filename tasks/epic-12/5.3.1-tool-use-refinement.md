# Story 5.3.1: Tool_use for Structured JSON Output

**Sprint:** 5 - Hybrid Questionnaire Generation Architecture
**Status:** pending
**Estimated:** 1 hour
**Prerequisite:** Story 5.3 complete

---

## Objective

Replace raw JSON mode with Claude's `tool_use` feature for more reliable structured output. Tool schemas give built-in validation and eliminate parsing logic.

---

## Prerequisites

- Story 5.3 complete (`QuestionnaireGenerationService` exists with parsing logic)
- Understanding of Claude's tool_use API

---

## Rationale

Claude's tool calling is specifically designed to produce valid JSON matching a schema. Using `tool_use`:
- Eliminates regex extraction of JSON from responses
- Removes handling of markdown code blocks and preambles
- Provides implicit schema validation
- Produces cleaner, more maintainable code

**Key Insight:** The existing `IClaudeClient.sendMessage()` already supports a `tools` option and returns `toolUse?: ToolUseBlock[]`. We should reuse this existing interface, NOT create a new `sendMessageWithTools()` method.

**Path:** `packages/backend/src/infrastructure/ai/tools/questionnaireOutputTool.ts`

```typescript
/**
 * Tool definition for structured questionnaire output
 *
 * Part of Epic 12.5: Hybrid Questionnaire Generation Architecture
 *
 * Claude calls this tool with the generated questionnaire,
 * ensuring the output matches QuestionnaireSchema exactly.
 * Using tool_use instead of raw JSON provides:
 * - Built-in schema validation
 * - No parsing/extraction logic needed
 * - More reliable structured output
 */

import { ALL_RISK_DIMENSIONS } from '../../../domain/types/QuestionnaireSchema.js';

/**
 * Build the tool definition dynamically to stay in sync with domain types
 */
export const questionnaireOutputTool = {
  name: 'output_questionnaire',
  description: 'Output the generated questionnaire in structured JSON format. Call this tool with the complete questionnaire data.',
  input_schema: {
    type: 'object',
    properties: {
      version: { type: 'string', const: '1.0' },
      metadata: {
        type: 'object',
        properties: {
          assessmentType: { type: 'string', enum: ['quick', 'comprehensive', 'category_focused'] },
          vendorName: { type: ['string', 'null'] },
          solutionName: { type: ['string', 'null'] },
          generatedAt: { type: 'string', format: 'date-time' },
          questionCount: { type: 'integer', minimum: 1 }
        },
        required: ['assessmentType', 'generatedAt', 'questionCount']
      },
      sections: {
        type: 'array',
        items: {
          type: 'object',
          properties: {
            id: { type: 'string' },
            title: { type: 'string' },
            riskDimension: {
              type: 'string',
              // Import from domain types - single source of truth
              enum: ALL_RISK_DIMENSIONS as unknown as string[]
            },
            description: { type: 'string' },
            questions: {
              type: 'array',
              items: {
                type: 'object',
                properties: {
                  id: { type: 'string' },
                  text: { type: 'string' },
                  category: { type: 'string' },
                  riskDimension: { type: 'string' },
                  questionType: { type: 'string', enum: ['text', 'yes_no', 'scale', 'multiple_choice'] },
                  required: { type: 'boolean' },
                  guidance: { type: 'string' },
                  options: { type: 'array', items: { type: 'string' } }
                },
                required: ['id', 'text', 'category', 'riskDimension', 'questionType', 'required']
              }
            }
          },
          required: ['id', 'title', 'riskDimension', 'description', 'questions']
        },
        minItems: 1
      }
    },
    required: ['version', 'metadata', 'sections']
  }
} as const;

/**
 * Type for the tool definition (for TypeScript imports)
 */
export type QuestionnaireOutputTool = typeof questionnaireOutputTool;
```

#### Change 2: Replace generate() method Claude call

**Important:** Use existing `sendMessage()` with `tools` option, NOT a new method.

```typescript
// BEFORE:
const response = await this.claudeClient.sendMessage(
  [{ role: 'user', content: prompt }],
  {
    systemPrompt: 'You are a healthcare AI governance expert. Generate questionnaires in valid JSON format only. Do not include any text outside the JSON object.',
  }
);

// Parse JSON from response
const schema = this.parseSchemaFromResponse(response.content);

// AFTER:
// Use existing sendMessage with tools option - IClaudeClient already supports this
const response = await this.claudeClient.sendMessage(
  [{ role: 'user', content: prompt }],
  {
    systemPrompt: 'You are a healthcare AI governance expert. Generate the questionnaire and call the output_questionnaire tool with the complete data.',
    tools: [questionnaireOutputTool],
    tool_choice: { type: 'tool', name: 'output_questionnaire' } // Force tool use
  }
);

// Extract schema from toolUse response (existing IClaudeClient returns this)
if (!response.toolUse || response.toolUse.length === 0) {
  throw new Error('Claude did not return tool_use response. Check IClaudeClient configuration.');
}

const toolUseBlock = response.toolUse.find(
  (block) => block.name === 'output_questionnaire'
);
if (!toolUseBlock) {
  throw new Error('Claude called wrong tool. Expected output_questionnaire.');
}

const schema = toolUseBlock.input as QuestionnaireSchema;
```

**Why reuse `sendMessage()`:**
- `IClaudeClient` already has `tools` support (see `packages/backend/src/application/interfaces/IClaudeClient.ts`)
- `ClaudeClient` already returns `toolUse: ToolUseBlock[]` in response (see `packages/backend/src/infrastructure/ai/ClaudeClient.ts`)
- Creating a new `sendMessageWithTools` would duplicate functionality
