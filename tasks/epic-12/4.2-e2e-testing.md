## Test Scenarios

### Scenario 1: Happy Path - Tool Flow

**Setup:**
- `USE_TOOL_BASED_TRIGGER=true`
- User logged in
- Assessment mode active

**Steps:**
1. User opens new conversation in assessment mode
2. User: "I need to assess an AI vendor for healthcare"
3. Claude: Asks clarifying questions
4. User: "It's Acme AI, they have a diagnostic tool"
5. Claude: "I can generate a questionnaire for you..."
6. User: "Yes, make it comprehensive"
7. **Expected:** Claude calls `questionnaire_ready` tool
8. **Expected:** "Generate Questionnaire" button appears
9. User clicks button
10. **Expected:** Claude generates questionnaire with markers
11. **Expected:** Download buttons appear

### Scenario 2: Happy Path - Fallback Flow

**Setup:**
- `USE_TOOL_BASED_TRIGGER=false`
- User logged in
- Assessment mode active

**Steps:**
1. User opens new conversation in assessment mode
2. User: "Generate a questionnaire"
3. **Expected:** Pattern matching detects trigger
4. **Expected:** Claude generates questionnaire immediately
5. **Expected:** Download buttons appear

### Scenario 3: Tool with Partial Info

**Setup:**
- `USE_TOOL_BASED_TRIGGER=true`

**Steps:**
1. User: "Generate a quick assessment"
2. **Expected:** Claude calls tool with only `assessment_type`
3. **Expected:** Button shows with null vendor/summary
4. User clicks button
5. **Expected:** Generation proceeds normally

### Scenario 4: No Tool Call for Questions

**Setup:**
- `USE_TOOL_BASED_TRIGGER=true`

**Steps:**
1. User: "What is in a questionnaire?"
2. **Expected:** Claude answers the question
3. **Expected:** Tool is NOT called
4. **Expected:** No button appears

### Scenario 5: Conversation Switch Clears State

**Steps:**
1. Trigger questionnaire_ready in conversation A
2. Button appears
3. Switch to conversation B
4. **Expected:** Button disappears
5. Switch back to conversation A
6. **Expected:** Button still gone (state was cleared)

## Monitoring Checklist

**Add to production monitoring to detect issues:**

### Log Sequence Verification

The happy path should produce logs in this sequence:
```
[ChatServer] Tool use detected: questionnaire_ready
[ChatServer] Emitted questionnaire_ready for conversation: conv-xxx
[ChatServer] Received generate_questionnaire for conversation: conv-xxx
[ChatServer] Emitted assistant_done with assessmentId: assess-xxx
[QuestionExtractionService] Extraction complete, emitting export_ready
```

### Alert Conditions

Set up alerts for:
- **Stuck state:** `questionnaire_ready` event without subsequent `generate_questionnaire` within 5 minutes (user saw button but didn't click)
- **Failed generation:** `generate_questionnaire` without `assistant_done` within 2 minutes
- **Tool call without emit:** Tool use logged but no WebSocket event emitted (backend error)

### Metrics to Track (before flag removal)

Record these to provide rollback data:
- Tool-based generations attempted (count)
- Tool-based generations completed successfully (count)
- Average time from `questionnaire_ready` to `generate_questionnaire` (user decision time)
- Error rate comparison: tool path vs. fallback path

## Automated E2E Test (Recommended for CI)

**Note:** Once tool-based flow is default (Sprint 5), bake Scenario 1 into CI to catch regressions.

If using Playwright or similar:

**Path:** `apps/web/e2e/questionnaire-generation.spec.ts`

```typescript
import { test, expect } from '@playwright/test';

/**
 * Epic 12 E2E Tests - Tool-Based Questionnaire Generation
 *
 * These tests should be added to CI once tool-based flow is default.
 * Run with: npx playwright test questionnaire-generation.spec.ts
 */

test.describe('Questionnaire Generation (Tool Flow)', () => {
  test.beforeEach(async ({ page }) => {
    // Login and setup
    await page.goto('/');
    // ... login steps (adjust based on auth implementation)
  });

  test('Scenario 1: tool flow shows generate button', async ({ page }) => {
    // Start conversation in assessment mode
    await page.click('[data-testid="mode-selector"]');
    await page.click('[data-testid="assessment-mode"]');

    // Send message that triggers tool
    await page.fill('[data-testid="chat-input"]', 'Yes, generate a comprehensive questionnaire');
    await page.press('[data-testid="chat-input"]', 'Enter');

    // Wait for and verify button appears (tool was called)
    await expect(page.locator('[data-testid="questionnaire-ready-card"]')).toBeVisible({ timeout: 15000 });
    await expect(page.locator('text=Comprehensive Assessment')).toBeVisible();
    await expect(page.locator('[data-testid="generate-questionnaire-btn"]')).toBeEnabled();
  });

  test('Scenario 1 continued: clicking button triggers generation', async ({ page }) => {
    // ... setup to show button (reuse from previous test or use fixtures)

    await page.click('[data-testid="generate-questionnaire-btn"]');

    // Verify loading state
    await expect(page.locator('text=Generating')).toBeVisible();

    // Wait for completion (questionnaire streams in)
    await expect(page.locator('text=QUESTIONNAIRE_START')).toBeVisible({ timeout: 60000 });

    // Verify download buttons appear
    await expect(page.locator('[data-testid="download-pdf"]')).toBeVisible({ timeout: 10000 });
  });

  test('Scenario 4: no tool call for questions about questionnaires', async ({ page }) => {
    // Start conversation in assessment mode
    await page.click('[data-testid="mode-selector"]');
    await page.click('[data-testid="assessment-mode"]');

    // Ask a question (should NOT trigger tool)
    await page.fill('[data-testid="chat-input"]', 'What questions are in a typical assessment?');
    await page.press('[data-testid="chat-input"]', 'Enter');

    // Wait for response
    await page.waitForSelector('[data-testid="assistant-message"]', { timeout: 15000 });

    // Verify NO generate button appeared
    await expect(page.locator('[data-testid="questionnaire-ready-card"]')).not.toBeVisible();
  });
});

//   retries: 1,      // Retry once on flaky Claude responses
// }
