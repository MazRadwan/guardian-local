**Sprint:** 2 - Infrastructure Layer
**Status:** complete

## Objective

Wire up the feature flag environment variable and update server initialization to inject `QuestionnaireReadyService` into `ChatServer`.

---

## Prerequisites

- Story 2.3 complete (ChatServer modifications)

---

## Implementation

### Step 1: Update Environment Files

**File:** `packages/backend/.env.example`

Add the feature flag:

```bash
# Feature Flags
# Set to 'true' to use Claude tool calls for questionnaire generation trigger
# Set to 'false' (default) to use pattern matching (TriggerDetection.ts)
USE_TOOL_BASED_TRIGGER=false
```

**File:** `packages/backend/.env` (local development)

```bash
# Enable for testing the new tool-based flow
USE_TOOL_BASED_TRIGGER=false
```

### Step 3: Add Feature Flag Logging

**File:** `packages/backend/src/index.ts`

Add logging on startup to indicate which mode is active:

```typescript
// Add after server starts
const useToolBasedTrigger = process.env.USE_TOOL_BASED_TRIGGER === 'true';
console.log(`[Server] Tool-based questionnaire trigger: ${useToolBasedTrigger ? 'ENABLED' : 'DISABLED (using pattern matching)'}`);
```

---

### Step 4: TypeScript Config Check

Ensure environment variable is typed (optional but recommended):

**File:** `packages/backend/src/types/env.d.ts` (create if doesn't exist)

```typescript
declare namespace NodeJS {
  interface ProcessEnv {
    // Existing vars...

    /**
     * Feature flag: Use Claude tool calls for questionnaire trigger
     * Must be exactly 'true' or 'false' (or undefined)
     */
    USE_TOOL_BASED_TRIGGER?: 'true' | 'false';
  }
}
```

**Note:** Using union type `'true' | 'false'` prevents accidental values like `'yes'` or `'1'`.
