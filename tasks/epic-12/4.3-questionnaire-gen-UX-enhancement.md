## Overview

Transform the current sticky footer "Generate Questionnaire" button into an inline chat card with dismiss/snooze functionality, localStorage persistence, and proper state transitions. The feature flag is being removed, so this becomes the sole questionnaire generation path.

---

## Current State

- `GenerateQuestionnaireButton` renders in sticky footer above Composer (ChatInterface.tsx:108-118)
- `pendingQuestionnaire` state is memory-only (lost on refresh)
- No dismiss/snooze capability
- State cleared on `assistant_done` (even for normal chat continuation)
- Embedded components (download, error) already render inline via `appendComponentToLastAssistantMessage`

---

## Requirements Summary

1. **Inline card placement** - Render in chat stream, not overlay
2. **Dismiss/snooze with persistence** - localStorage, survives refresh
3. **State transitions** - Ready → Generating → Download Ready (or Error)
4. **Intent handling** - Chat continuation shouldn't clear pending unless new tool call
5. **Slim sticky affordance** - When card scrolled out of view

## User Decisions

| Decision | Choice | Implication |
|----------|--------|-------------|
| Download UI | **Unified card** | Download buttons in same inline card, not embedded in message |
| Snooze duration | **Until new tool call** | No timer - dismissed stays dismissed until Claude re-triggers |
| Payload persistence | **Yes, persist** | `pendingQuestionnaire` saved to localStorage, survives refresh |

---

## Recommended Approach: Hybrid (Minimal + Clean UX)

### Why Hybrid?

| Aspect | Minimal | Clean UX | **Hybrid** |
|--------|---------|----------|------------|
| Component placement | Embedded in messages | Slot in MessageList | **Slot in MessageList** |
| State management | Existing fields | New state machine | **Existing + minimal additions** |
| Sticky indicator | Not included | Full component | **Simple sticky affordance** |
| History replay handling | Risky (ghost cards) | Robust | **Robust (slot approach)** |
| Complexity | Low | High | **Medium** |

The slot approach (from Clean UX) is safer than embedding in messages because:
- Old messages from history won't render stale cards
- Single source of truth for card visibility
- Easier to manage transitions

## State Clearing Rules

| Trigger | Clear Payload | Clear UIState | Clear localStorage |
|---------|---------------|---------------|-------------------|
| User clicks Dismiss | No | → hidden | Set dismiss flag |
| User downloads successfully | Yes | → hidden | Clear payload |
| User closes Download card | Yes | → hidden | Clear payload |
| Conversation switch | Yes | → hidden | Don't clear (persisted per conversation) |
| New `questionnaire_ready` tool call | Yes (replace) | → ready | Clear dismiss, save new payload |
| Normal `assistant_done` | **NO** | **NO** | **NO** |
| `assistant_done` after generation | No (keep for download) | → download | Keep payload |
| Logout | N/A | N/A | Clear ALL for user |

## State Machine

```
                                  ┌─────────────────────────────────┐
                                  │                                 │
  ┌───────────questionnaire_ready─┼──────────────────────┐          │
  │  (new tool call clears dismiss│& old payload)        │          │
  │                               │                      ▼          │
  │                               │                   READY ────────┤
  │                               │                     │           │
  │   ┌───────────────────────────┘      user clicks    │           │
  │   │                                  "Generate"     │           │
  │   │                                      │          │           │
  │   │                                      ▼          │           │
  │   │                               GENERATING        │           │
  │   │                                   │             │ user      │
  │   │                    export_ready   │             │ dismisses │
  │   │                          │        │             │           │
  │   │                          ▼        │             ▼           │
  │   │                      DOWNLOAD ◄───┘         DISMISSED       │
  │   │                          │                  (permanent      │
  │   │                          │                   in localStorage│
  │   │    extraction_failed     │                   until new      │
  │   │          │               │                   tool call)     │
  │   │          ▼               │                      │           │
  │   │       ERROR              │                      │           │
  │   │          │               │                      │           │
  │   │   user retries           │                      │           │
  │   │          │               │                      │           │
  │   └──────────┴───────────────┴──────────────────────┘           │
  │                                                                 │
  │               user downloads or explicitly closes card          │
  │◄────────────────────────────────────────────────────────────────┘
  │
HIDDEN
```

**Key behaviors:**
- **Dismiss is permanent** until Claude issues a new `questionnaire_ready` tool call
- **Payload persists** in localStorage - card survives page refresh
- **Unified card** - same card transitions through Ready → Generating → Download
- **New tool call** clears both dismiss state AND old payload (replaces with new)
