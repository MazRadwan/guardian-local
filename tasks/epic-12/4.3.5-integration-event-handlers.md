### 2. useWebSocketEvents.ts - Fix Event Handlers

#### handleQuestionnaireReady
```typescript
const handleQuestionnaireReady = useCallback((payload: QuestionnaireReadyPayload) => {
  // Clear any dismiss state - new tool call supersedes snooze
  persistence.clearDismiss(payload.conversationId);

  // Save payload to localStorage for persistence across refreshes
  persistence.savePayload(payload.conversationId, payload);

  // Update store
  useChatStore.getState().setPendingQuestionnaire(payload);
  useChatStore.getState().setQuestionnaireUIState('ready');
  useChatStore.getState().setQuestionnaireError(null);
}, [persistence]);
```

## State Clearing Rules Reference

| Trigger | Clear Payload | Clear UIState | Clear localStorage |
|---------|---------------|---------------|-------------------|
| User clicks Dismiss | No | → hidden | Set dismiss flag |
| User downloads successfully | Yes | → hidden | Clear payload |
| User closes Download card | Yes | → hidden | Clear payload |
| Conversation switch | Yes (store) | → hidden | Don't clear |
| New `questionnaire_ready` | Yes (replace) | → ready | Clear dismiss, save new |
| Normal `assistant_done` | **NO** | **NO** | **NO** |
| `assistant_done` after gen | No | → download | Keep payload |
| Logout | N/A | N/A | Clear ALL for user |
