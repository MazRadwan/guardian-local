# Story 39.1.1: RegexResponseExtractor Core + Text Preprocessing

## Description

Create the core regex extraction engine that parses Guardian-generated questionnaire documents into structured Q&A pairs. This replaces the ~5 minute Claude API call for extraction. The extractor uses anchored patterns (`^Question\s+\d+\.\d+\s*$` and `^Response:\s*$`) that match the exact format produced by `WordExporter.ts`.

Also create a `TextPreprocessor` module that cleans raw text before regex extraction: strips Guardian footer, pdf-parse page markers, and normalizes whitespace.

## Acceptance Criteria

- [ ] `RegexResponseExtractor.extract()` parses raw text into `ExtractedResponse[]`
- [ ] Uses multiline `^Question\s+(\d+)\.(\d+)\s*$` pattern (start-of-line anchored)
- [ ] Uses multiline `^Response:\s*$` to split question text from answer text
- [ ] Extracts assessmentId via `/Assessment\s+ID:\s*\n?\s*([a-f0-9-]{36})/i`
- [ ] Extracts vendor name from first non-empty line after "AI Vendor Assessment Questionnaire"

- [ ] Post-processing strips trailing section headers from response text

- [ ] Empty responses get `confidence: 0.5` (vendor may have skipped)
- [ ] Returns `{ assessmentId, vendorName, responses, parseTimeMs }`
- [ ] Under 300 LOC per file

## Technical Approach

### 1. Create TextPreprocessor

**File:** `packages/backend/src/infrastructure/extraction/TextPreprocessor.ts`

Handles pre-processing of raw text before regex extraction:
- Strip all pdf-parse page markers: `rawText.replace(/--\s*\d+\s*of\s*\d+\s*--/g, '')`
- Strip Guardian footer: from "Generated by Guardian AI Vendor Assessment System" to end
- Normalize excessive whitespace: `\n{3,}` to `\n\n`
- Trim leading/trailing whitespace

### 2. Create RegexResponseExtractor

**File:** `packages/backend/src/infrastructure/extraction/RegexResponseExtractor.ts`

Core algorithm:
1. Call `TextPreprocessor.preprocess(rawText)`
2. Extract assessmentId and vendorName from header
3. Find all `Question X.Y` markers with character positions (multiline regex)
4. For each marker, extract text block between it and next marker (or end of document)
5. Within each block, split on `^Response:\s*$` (multiline)
6. Everything before `Response:` = question text (strip leading section headers)
7. Everything after `Response:` = answer text (strip trailing section headers)
8. Return `ExtractedResponse` array with sectionNumber, questionNumber, questionText, responseText

**Key patterns (from feasibility test):**
```typescript
const QUESTION_MARKER = /^Question\s+(\d+)\.(\d+)\s*$/gm;
const RESPONSE_MARKER = /^Response:\s*$/m;
const SECTION_HEADER = /^Section\s+\d+:\s+[^\n]+$/m;
const ASSESSMENT_ID = /Assessment\s+ID:\s*\n?\s*([a-f0-9-]{36})/i;
```

### 3. Return Type

```typescript
export interface RegexExtractionResult {
  assessmentId: string | null;
  vendorName: string | null;
  responses: Array<{
    sectionNumber: number;
    questionNumber: number;
    questionText: string;
    responseText: string;
    confidence: number;       // 1.0 for text, 0.5 for empty
    hasVisualContent: boolean; // always false here; set by DocxImageDetector
  }>;
  parseTimeMs: number;
}
```
